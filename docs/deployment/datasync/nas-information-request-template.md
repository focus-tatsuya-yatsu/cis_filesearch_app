## 📋 受領情報チェックリスト

クライアントから情報を受け取ったら、以下をチェック：

```
NAS基本情報:
  ✅ NASベンダー/型番
  ✅ IPアドレスまたはホスト名
  ✅ プロトコル（SMB or NFS）

SMB情報（該当する場合）:
  ✅ 共有名
  ✅ ドメイン名
  ✅ ユーザー名
  ✅ パスワード（セキュア受領）
  ✅ 同期対象フォルダパス

NFS情報（該当する場合）:
  ✅ NFSエクスポートパス
  ✅ NFSバージョン
  ✅ 同期対象フォルダパス

ネットワーク情報:
  ✅ ネットワークセグメント
  ✅ VPN要否
  ✅ ファイアウォール設定情報

データ情報:
  ✅ ファイル数（概算）
  ✅ データ総量（概算）
  ✅ 除外フォルダリスト

セキュリティ:
  ✅ 暗号化要件確認
  ✅ アクセス制御要件確認
```

## 🔒 セキュリティ注意事項

### パスワード受領方法

❌ **避けるべき方法**:
- 平文メール
- Slackなどのチャットツール
- パスワードなしのExcelファイル

✅ **推奨方法**:
1. **電話受領**: 口頭で聞き取り、記録
2. **パスワード付きZIP**: パスワードは別メールまたは電話
3. **AWS Secrets Manager**: 直接入力を依頼
4. **セキュアファイル転送**: Box、OneDrive（暗号化）

### 受領後の保管

```
パスワード等の機密情報は以下に保管:
  ✅ AWS Secrets Manager（推奨）
  ✅ パスワード管理ツール（1Password、LastPass等）
  ❌ .envファイル（Git管理外でも推奨せず）
  ❌ テキストファイル
```

## 📞 電話ヒアリングスクリプト

メールで情報収集が困難な場合、電話でヒアリングする際のスクリプト：

```
【冒頭】
お世話になっております。CIS File Searchの△△です。
NAS接続情報の件でお電話させていただきました。
今、10分ほどお時間よろしいでしょうか？

【本題】
それでは、順番にご確認させてください。

1. NASのメーカーと型番を教えていただけますか？
   （メモ: ___________________________）

2. NASのIPアドレスは何番でしょうか？
   （メモ: ___________________________）

3. ファイル共有のプロトコルですが、
   Windowsのネットワークドライブでアクセスされていますか？
   （はい → SMB、いいえ → NFS）

4. （SMBの場合）共有名は何という名前でしょうか？
   例えば \\192.168.1.100\○○ の○○部分です。
   （メモ: ___________________________）

5. 接続に使用するユーザー名とパスワードを教えてください。
   （メモ: User: ____________ / Pass: ____________）

6. 同期したいフォルダのパスを教えてください。
   （メモ: ___________________________）

【確認】
復唱させていただきます。
- NAS: （型番）
- IP: （IPアドレス）
- 共有名: （共有名）
- ユーザー: （ユーザー名）
- フォルダ: （フォルダパス）

こちらで間違いございませんでしょうか？

【締め】
ありがとうございました。
それでは、この情報でDataSyncの設定を進めさせていただきます。
設定完了後、テスト実施の際に再度ご連絡いたします。
```
